# Return Inventory

Returning inventory allows users to **place issued items back into the system** while automatically updating inventory counts and associated data fields. This process applies to **semi-consumable and durable items** and guides users to the correct cabinet and drawer location for placement.

**Also known as:** *Return Items*, *Return Transaction*

---


### Return Items
- Used for **semi-consumable or durable items** that have been used at least once
- Items are returned as **previously used** and available for future use
- Credits back any debits against recorded data fields

How an item is handled depends on how it was configured in the system database.

---

## How to Return Items

### 1. Log In

1. Sign in using the credentials assigned to you.
2. Enter your information and click **Submit**.

After logging in, you will see the **front page of your site**, where all items available at your station are displayed.

---

### 2. Access Return Inventory

- Users may return items only if **Return rights** have been assigned
- Click the **Return** link to continue

---

### 3. Select a Return Transaction

- A list of all **issued semi-consumable and durable items** is displayed
- Select the transaction you want to return
- Enter the **quantity** to be returned

**Note:**
- If you are an administrator, a **Show All** button is available to view items issued to all employees
- If multiple identical items exist, ensure you are returning the correct transaction for the associated data fields

---

### 4. Return Options

- Click **Return** to place the item back into inventory
- **Return and Re-Issue** is available to return the item and immediately issue it to another user
- Semi-consumable items may have an option to **Return or Scrap**, depending on condition
  - Scrapping an item issues it out of inventory
- Durable items cannot be scrapped and must be fully issued or returned through a physical transaction

---

## Cabinet and Drawer Access

- The cabinet unlocks automatically
- Item quantity and location are displayed, similar to the checkout process
- A **bird’s-eye drawer view** shows the exact bin location for item placement

Once the item is placed back into the drawer and the drawer is closed:
- The cabinet locks automatically
- The user is logged out of the system
